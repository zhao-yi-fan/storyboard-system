# Storyboard Shot Media Fields

## User goal

让镜头编辑和生成流程区分清楚：

- 哪些字段是镜头的共用叙事信息
- 哪些字段只影响视频生成
- 首帧图是否作为视频生成输入应由用户显式控制

避免把不适合首帧图的字段误塞进图像 prompt，导致首帧图质量下降或语义跑偏。

## Current problem

- 当前“镜头详情”把首帧和视频相关字段混在一起展示。
- 生成首帧时仍会展示或暗示使用机位、镜头运动、时长等视频特有信息。
- 生成视频默认强依赖首帧图，没有显式开关。
- 生成弹窗没有根据任务类型区分字段，用户容易误解。

## Field grouping

### 共用字段

这些字段用于镜头的叙事描述，同时可被首帧和视频共同使用：

- 角色
- 背景场景
- 画面描述
- 台词
- 情绪
- 风格预设
- 风格补充
- 备注

### 视频特有字段

这些字段主要影响视频生成，不应该默认参与首帧 prompt：

- 景别
- 机位
- 镜头运动
- 时长
- 是否使用当前首帧

## Generation behavior

### 生成首帧

- 使用共用字段构造 prompt
- 不将景别、机位、镜头运动、时长直接写入首帧 prompt
- 首帧本质上是“视频首帧图 / 关键帧图”，但它仍然是图片任务

### 生成视频

- 使用共用字段 + 视频特有字段构造 prompt
- 新增 `use_first_frame` 开关
- `use_first_frame=true`
  - 如果当前有首帧图，则作为首帧输入
  - 如果当前没有首帧图，则后端先自动生成首帧，再继续视频任务
- `use_first_frame=false`
  - 直接走文生视频
  - 不依赖当前镜头首帧

## UI changes

### 镜头详情表单

分成两个区块：

- 共用信息
- 视频信息

### 生成首帧弹窗

只显示与首帧相关的字段：

- 场景标题
- 地点
- 时间
- 背景场景
- 角色
- 画面描述
- 情绪
- 风格预设
- 风格补充
- 台词
- 备注

不显示：

- 景别
- 机位
- 镜头运动
- 时长

### 生成视频弹窗

分成：

- 共用字段
- 视频字段
- 首帧输入状态

并显示 `use_first_frame` 当前状态。

## API changes

### 视频预览

`GET /api/storyboards/:id/video-generation-preview`

新增 query：

- `use_first_frame=true|false`

### 视频生成

`POST /api/storyboards/:id/generate-video`

新增 body：

- `use_first_frame: boolean`

## Acceptance criteria

- 生成首帧时不会再把视频特有字段作为主要输入展示给用户
- 生成视频时用户可以显式关闭“使用当前首帧”
- 关闭首帧后，后端不再强制依赖当前首帧图
- 首帧弹窗和视频弹窗展示字段分组清楚
