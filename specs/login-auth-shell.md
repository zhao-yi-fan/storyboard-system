# 登录页与退出登录壳层

## 用户目标

为分镜系统提供一个统一的登录入口和退出登录入口，让未登录用户先进入登录页，登录后才能访问项目页、工作台、资产库和新建项目页。

## 工作流

1. 未登录用户访问任意业务页时，重定向到 `/login`。
2. 用户在登录页输入账号和密码，点击登录。
3. 前端创建本地会话并跳转到 `/projects`。
4. 已登录用户访问 `/login` 时，自动跳转到 `/projects`。
5. 用户在顶部账户菜单点击 `退出登录` 后，清除本地会话并回到 `/login`。

## 数据形状

本期只做前端壳层登录，不接真实后端鉴权。

本地会话结构：

```ts
type AuthSession = {
  account: string;
  displayName: string;
  roleLabel: string;
};
```

存储位置：
- `localStorage.storyboard_auth_session`

## UI 规则

- 登录页采用视频背景。
- 视频地址由前端环境变量提供：
  - `VITE_LOGIN_BG_VIDEO_URL`
- 未配置视频地址时，登录页使用深色渐变背景回退。
- 登录页右侧保留登录卡片，内部表单行为以现有 Figma Make 方案为基线。
- 所有业务页顶部右侧显示统一账户菜单：
  - 用户名
  - 角色标签
  - 账号设置（占位）
  - 退出登录

## 失败与重试

- 账号或密码为空：阻断并显示错误提示。
- 若未配置 `VITE_LOGIN_BG_VIDEO_URL`：不报错，直接使用静态深色背景。
- 本期不校验真实账号密码，不返回后端错误。

## 验收标准

1. 未登录访问 `/projects`、`/workspace`、`/assets`、`/import` 时会跳转到 `/login`。
2. 登录成功后进入 `/projects`。
3. 已登录访问 `/login` 会跳回 `/projects`。
4. 顶部账户菜单可执行退出登录。
5. 退出登录后再次访问业务页会回到 `/login`。
6. 前端构建通过。
