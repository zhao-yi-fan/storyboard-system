# 角色主设定图改为 Seedream 图生图

## 用户目标

为每个角色生成更稳定、更细节化的主设定图，减少纯文生图导致的人脸、发饰、首饰、服装细节漂移。

## 工作流

1. 在角色资产里上传或填写一张 `角色参考图`。
2. 点击 `生成主设定图`。
3. 后端使用 `Seedream` 图生图：
   - 角色参考图作为人物参考输入。
   - 若配置了系统版式参考图，则一并作为参考输入。
4. 生成结果保存到 `design_sheet_url`。

## 数据结构

角色图片字段只保留：

- `avatar_url`
  - 仅用于主设定图生成时的人物参考图
- `design_sheet_url`
  - 角色正式展示图和后续参考图

不再使用：

- `avatar_preview_url`
- `design_sheet_preview_url`

## UI 规则

- 资产库角色详情分成两块：
  - `角色参考图`
  - `角色主设定图`
- `角色参考图` 只说明“用于生成主设定图”，不作为正式展示主图。
- `角色主设定图` 才是角色对外展示结果。
- 没有 `角色参考图` 时，`生成主设定图` 应阻断并提示。

## Prompt 规则

主设定图 prompt 必须明确强调：

- 面部细节
- 发型与发饰
- 耳饰 / 项链 / 腰饰
- 服装结构与花纹
- 面料材质
- 配色方案
- 正面 / 侧面 / 背面三视图
- 线稿辅助区
- 身高比例参考
- 允许少量中文栏目标题和标注

## 失败与重试

- 无 `avatar_url`：直接失败，提示先上传角色参考图。
- Seedream 未配置：直接失败，提示缺少 `SEEDREAM_IMAGE_API_KEY`。
- OSS 下载或保存失败：保留错误信息，不吞掉异常。

## 验收标准

1. 角色无参考图时，无法生成主设定图。
2. 角色有参考图时，主设定图生成固定走 `Seedream` 图生图。
3. 前后端不再依赖任何 `*_preview_url` 角色图片字段。
4. `avatar_url` 不再参与角色正式展示图逻辑。
5. `design_sheet_url` 继续作为角色正式展示图和后续参考图。
